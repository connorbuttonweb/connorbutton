The Website Project

My self-built portfolio site developed with HTML, Python, CSS, and AI tools to showcase my projects and personal work.

This website began as a blank canvas as a personal challenge to see how far I could go without relying on drag-and-drop builders or expensive subscriptions. I started with zero web development experience and used AI tools, curiosity, and persistence to bring it to life. Every section, feature, and bug fix taught me something new about design, code, and creative problem-solving. More than just a portfolio, this project reflects my values: practical learning, doing things the hard way when it’s worth it, and proving that initiative and resourcefulness go a long way. Below are some of the many files I have created throughout this project. I have chosen each one to highlight something different and specific for a variety of reasons.

index.hmtl

First when looking at index.html I would like to highlight the standard html code that is used to build the general website. Using comments in the code I labeled the various sections. Most of the sections are fairly standard using < div > and < p > to create sections of code and text with the exception of the gallery section which uses scripts. Another thing that index.html highlights is the different ways the user is redirected to new sites. This is shown in #three when redirecting to the project pages. Another example of this is the use of the free website formspree.com which filters the contact section for spam and redirects emails to my personal email account.
